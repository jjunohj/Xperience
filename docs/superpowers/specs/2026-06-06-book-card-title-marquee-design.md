# BookCard 제목 마퀴 슬라이드 설계

이슈: #56

## 배경

Book 페이지 `BookCard`의 긴 제목은 현재 `line-clamp-1`(`...`)로 잘린다.
저자(`shrink-0`)는 우측 자연 너비를 유지하고 제목이 남은 폭을 쓰는 레이아웃은
유지하되, 잘림 대신 **넘칠 때만 슬라이드**해 전문을 보여주고 싶다.

## 결정 사항

- **트리거**: 제목이 남은 폭보다 넓을 때만 자동 재생. 짧은 제목은 정지.
- **복귀**: 끝까지 슬라이드 → 3초 정지 → 첫 위치로 **즉시 점프** → 반복.
- **속도**: 일정 px/초(기본 30px/s). 시작 정지 1s, 끝 정지 3s. 모두 조정 가능.
- **분리**: 마퀴 로직을 `MarqueeText` 컴포넌트로 분리(단독 이해/재사용 가능).
- **엔진**: 기존 의존성 framer-motion. `x` 키프레임 + `repeat:"loop"`로
  루프 경계의 즉시 점프를 자연스럽게 표현.
- **접근성**: `prefers-reduced-motion`이면 애니메이션 없이 말줄임 폴백.

## SEO / SSG / SSR 안전장치 (필수)

- 제목은 **진짜 `<h3>` 시맨틱 헤딩**으로 유지.
- 텍스트는 일반 children으로 렌더 → **정적 HTML에 항상 존재**(JS 주입 금지).
- 첫 페인트는 정적(말줄임)으로 렌더, `useEffect` 측정 후에만 애니메이션 →
  **하이드레이션 미스매치 없음**.
- `overflow-hidden` + transform 전용 애니메이션 → **CLS 0**.
- 단일 텍스트(복제 없음) → 중복 콘텐츠·스크린리더 이중 낭독 없음.

## 측정 방식

컨테이너 `scrollWidth - clientWidth` 하나로 넘침 거리 계산.
transform은 scrollWidth에 영향을 주지 않아 정지/마퀴 두 분기에서 일관.
`ResizeObserver` + `document.fonts.ready`로 리사이즈·폰트 로드 후 재측정.

## 범위 밖 (YAGNI)

- 호버 일시정지, 연속 무한 루프(텍스트 복제), 방향 토글.
- 테스트 프레임워크 도입(프로젝트에 테스트 러너 없음 → 타입체크·lint·시각 검증).
